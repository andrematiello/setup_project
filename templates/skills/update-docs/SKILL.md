# Update Docs

## Descrição

Ative quando uma implementação foi concluída e o agente está prestes a apresentar o resultado final.

## Procedimento

1. Identifique quais arquivos foram criados, alterados ou removidos.
2. Mapeie o tipo de mudança para os docs corretos: [ADAPTÁVEL — ajuste os mapeamentos ao seu projeto]
   - nova tabela ou coluna → doc de schema/modelo de dados
   - nova rota, página ou função de backend → doc de arquitetura
   - nova regra de negócio → doc de domínio de negócio
   - nova integração → doc de integrações ou infraestrutura
   - nova feature → doc de features
3. Atualize os documentos necessários antes de apresentar o resultado.
4. Verifique se nomes de arquivos, funções, tabelas e rotas batem com o código real.
5. Informe ao usuário quais docs foram atualizados.

## Nunca fazer

- Nunca apresentar implementação concluída sem revisar docs afetados.
- Nunca documentar intenção futura como fato implementado.
- Nunca manter doc desatualizado quando a mudança alterou comportamento real.
