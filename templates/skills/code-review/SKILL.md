# Code Review

## Descrição

Ative quando o pedido contiver termos como revisar, melhorar, refatorar, está bom, o que acha, tem problema ou pode melhorar.

## Procedimento

1. Analise o escopo do pedido e leia os arquivos relevantes.
2. Revise pelas dimensões na ordem de prioridade: segurança, correção, qualidade, performance e consistência.
3. Registre achados com severidade, localização, problema e solução.
4. Use `.claude/commands/review.md` como formato base do relatório.
5. Termine com um sumário executivo e recomendação final.

## Nunca fazer

- Nunca começar a resposta por resumo antes dos achados.
- Nunca tratar problema de segurança como sugestão cosmética.
- Nunca fugir do formato de severidade e prioridade do review.
